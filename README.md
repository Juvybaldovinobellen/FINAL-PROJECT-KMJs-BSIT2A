## Project Description
The BU Transakto– Registrar System is a 
comprehensive web-based platform designed to streamline and digitize the management of school 
document requests. Traditionally, students and registrar staff rely on manual processes, paper 
forms, and in-person visits to submit, process, and track document requests, which often leads to 
delays, miscommunication, and inefficiency. This system aims to address these challenges by 
providing a centralized, user-friendly, and secure platform where both students and staff can 
perform their tasks efficiently. For students, the system allows them to submit various school 
document requests online, such as transcripts, certificates of enrollment, and other official records, 
eliminating the need for repeated physical visits to the registrar’s office. Students can also monitor 
the processing status of their requests in real-time, view their history of past requests, and provide 
feedback or reviews on the service received. For registrar staff, the system provides a dashboard 
to efficiently manage incoming requests, update the status of each request, and ensure timely 
completion. Staff can also create and manage document templates, review student feedback, and 
configure system settings to match the institution’s operational requirements. By digitizing these 
processes,  the  platform  reduces  manual  paperwork,  improves  transparency,  and  enhances 
communication between students and the registrar’s office. The system is designed to be scalable, 
secure, and accessible from multiple devices, supporting both desktop and mobile use, ensuring 
that users can access services anytime and anywhere. Ultimately, this platform aims to improve 
administrative  efficiency,  student  satisfaction,  and  overall  workflow  management  within  the 
school environment.

# The project demonstrates our understanding of:
- **Full-Stack Architecture:** 
Building BU Transakto using Bootstrap frontend, Node.js/Express backend, and MongoDB database.

- **RESTful API Design:** Implementing CRUD operations for document requests, templates, and user management.

- **Database Modeling:** 
Creating Mongoose schemas for users, document requests, templates, and feedback.

- **UI/UX Design:** Designing separate intuitive interfaces for students and registrar staff.

- **Business Logic:** Automating request tracking, status updates, and notification workflows.

- **Authentication & Security:** Secure login, registration, role-based access control (Student/Staff), and session handling.

- **Group Collaboration:** Using GitHub for teamwork and code management.
- **System Documentation:** Creating ERD, DFD, and Use Case diagrams.

## Weekly Development Roadmap
### Phase 1 - Project Planning (Week 10)
Proposal drafting, system architecture, feature list, database schema (ERD), GitHub repo setup, initial group coordination.

### Phase 2 - Backend Setup (Week 12)
Server setup using Node.js and Express; API scaffolding; MongoDB/Mongoose schema creation based on Phase 1 planning.

### Phase 3 - Frontend Design (Week 12)
Development of interface layouts using HTML, CSS, and Bootstrap based on Week 10 design; home, about, and basic navigation setup.

### Phase 4 - Form Submission & Data Insertion (Week 13)
Creating frontend forms and integrating them with backend APIs; validate and store data in MongoDB.

### Phase 5 - Data Retrieval & Display (Week 14)
Fetching and displaying MongoDB data using JavaScript and Bootstrap UI; basic dashboards and listings.

### Phase 6 - Update & Delete Functionality (Week 15)
Implement PUT and DELETE operations; integrate modals or pages for editing and removing data; final CRUD completion.

### Phase 7 - Project Polishing & Documentation (Week 16)
Testing, debugging, UI/UX enhancements, deployment on Render, README writing, documentation and preparation for final defense.
